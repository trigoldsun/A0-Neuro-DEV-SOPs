### 1.1 架构说明

本平台采用**单机多容器**架构，通过 Docker Compose 编排所有服务。多个研发项目共享基础设施（数据库、缓存、对象存储），各项目通过独立的数据库、Redis命名空间和MinIO桶实现数据隔离。

平台服务分为两个层级：

| 层级 | 说明 | 运行模式 |
|:---|:---|:---|
| **Tier1 基础设施** | 数据库、缓存、网关、监控等共享组件 | 常驻运行 |
| **Tier2 项目服务** | 各项目的微服务应用 | 按需启停 |

## 3. 快速开始（5分钟上手）

1. 获取 SSH 私钥文件（联系平台管理员）
2. 安装 SSH 客户端
3. （可选）安装 DBeaver / RedisInsight 等GUI工具

### 3.2 连接服务器

```bash
# 将私钥放到 ~/.ssh/ 目录

# SSH 连接

```

### 3.3 查看平台状态

```bash

platformctl resources
```

### 3.4 创建你的项目

```bash

platformctl project list
platformctl project start w001
platformctl project stop w001
```

## 4. 项目入驻指南

### 4.1 创建新项目

每个新项目通过 `create-project.sh` 脚本自动化入驻，脚本会自动完成：

1. 分配100个端口（30000起步，每项目+100）
2. 创建独立的 PostgreSQL 数据库 + 用户
3. 创建独立的 Redis ACL 用户（带key前缀隔离）
4. 创建独立的 MinIO 存储桶
5. 生成项目目录和 docker-compose.yml 模板
6. 将凭证写入 `.env.secrets`

```bash
platformctl project create <项目编号> <项目描述名>

# 示例
platformctl project create w001 user-service
platformctl project create w002 order-service
```

### 4.2 项目目录结构

```
projects/
└── w001-user-service/
    ├── docker-compose.yml    # 项目服务编排（已预填基础设施连接信息）
    ├── Dockerfile            # Go/Gin 多阶段构建模板
    ├── configs/              # 项目自定义配置
    └── db-migrations/        # 数据库迁移脚本
```

### 4.3 部署你的代码

```bash
cd /opt/neuro-devsituat-platform/projects/w001-user-service
git clone <你的仓库地址> .
# 修改 docker-compose.yml
docker compose up -d
```

### 4.4 配置 API 网关路由

编辑 `/opt/neuro-devsituat-platform/configs/nginx/nginx.conf`：

```nginx
upstream w001_user_service {
    server w001-user-service:30000;
}

# 在 server { } 块中添加
location /api/v1/user/ {
    proxy_pass http://w001_user_service/api/v1/user/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

重载：`docker exec gateway-nginx nginx -s reload`

## 5. 日常运维命令

### 5.1 platformctl 管理工具

```bash
platformctl infra start|stop|restart|status   # 基础设施管理
platformctl project create <编号> [名称]       # 创建新项目
platformctl project start|stop <编号>          # 启停项目
platformctl project list                       # 列出所有项目
platformctl status                             # 全平台状态
platformctl resources                          # 资源报告
platformctl backup                             # 手动备份
```

### 5.2 常用 Docker 命令

```bash
docker ps                                      # 运行中容器
docker logs <容器名> --tail 100 -f              # 查看日志
docker exec -it <容器名> sh                     # 进入容器
docker restart <容器名>                         # 重启容器
docker stats --no-stream                       # 资源占用
```

### 5.3 常用数据库操作

```bash
docker exec postgresql18 psql -U platform_admin -c "\l"    # 列出数据库
docker exec -it postgresql18 psql -U w001_user -d w001_db  # 连接项目库
docker exec -i postgresql18 psql -U w001_user -d w001_db < migrate.sql  # 执行SQL
```

### 5.4 Redis 操作

```bash
docker exec -it redis7 redis-cli -a <密码>                 # 连接
docker exec redis7 redis-cli -a <密码> info memory          # 内存信息
```

### 5.5 MinIO 操作

浏览器打开 `http://101.133.168.46:20005` 管理控制台。

## 6. 数据库版本控制

遵循《一号文件》，**禁止手动直连改库**。必须使用代码化迁移工具（Goose / golang-migrate）。

```bash
# Goose 示例
cd /opt/neuro-devsituat-platform/projects/w001-xxx/db-migrations
goose create create_users_table sql
docker exec -i postgresql18 psql -U w001_user -d w001_db < 001_create_users_table.sql

# golang-migrate 示例
migrate -path db-migrations/ -database "postgres://w001_user:<密码>@101.133.168.46:20002/w001_db?sslmode=disable" up
```

## 8. 故障排查

```bash
# 服务无法启动
docker ps -a
docker logs <容器名> --tail 50
free -h && df -h && docker stats --no-stream

# 数据库连接失败
docker exec postgresql18 pg_isready -U platform_admin
docker exec postgresql18 psql -U platform_admin -c "\l"

# Redis连接失败
docker exec redis7 redis-cli -a <密码> ping    # 应返回 PONG

# 磁盘不足
df -h && docker system prune -a -f

# 端口冲突
ss -tlnp | grep <端口号>
```

## 9. 安全注意事项

1. **凭证管理**: 密码存储在 `.env.secrets`（权限600），禁止明文写入代码
2. **环境隔离**: 严禁接入生产流量或存储生产敏感数据
3. **端口规范**: 基础设施 20000-29999，项目服务 30000-49999
4. **资源限制**: 每个容器设有CPU和内存上限
5. **数据备份**: PostgreSQL每日凌晨2点自动备份，保留7天
6. **日志轮转**: Docker容器日志自动轮转（单容器上限30MB）

## 附录C: Dockerfile 模板

```dockerfile
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o main .

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/main .
EXPOSE 8080
USER nobody
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --spider -q http://127.0.0.1:8080/health || exit 1
ENTRYPOINT ["./main"]
```
