### 1.1 架构说明

本平台采用**单机多容器**架构，通过 Docker Compose 编排所有服务。多个研发项目共享基础设施（数据库、缓存、对象存储），各项目通过独立的数据库、Redis命名空间和MinIO桶实现数据隔离。

平台服务分为两个层级：

| 层级 | 说明 | 运行模式 |
|:---|:---|:---|
| **Tier1 基础设施** | 数据库、缓存、网关、监控等共享组件 | 常驻运行 |
| **Tier2 项目服务** | 各项目的微服务应用 | 按需启停 |

## 4. 项目入驻指南

### 4.0 重要：资源登记制度

#### 为什么要登记？

多个团队共享同一套基础设施（端口、路由、数据库等），若不登记必然冲突。本制度确保所有资源占用信息公开透明，杜绝团队间信息孤岛。

#### 适用范围

| 资源类型 | 对应附表 | 冲突后果示例 |
|:---|:---|:---|
| 宿主机端口 (30000-49999) | 附录A | 服务启动失败，端口被占用 |
| API网关路由路径 | 附录B | 请求被错误转发到其他服务 |
| Docker容器名前缀 | 附录C | 容器名冲突无法启动 |
| Docker网络名 | 附录D | 网络创建失败 |
| PostgreSQL数据库名 | 附录E | 数据库创建失败 |
| MinIO桶名 | 附录F | 桶创建失败 |

#### 谁需要登记？

每个开发团队成员，在以下任一场景发生前必须先登记：

- 新增一个微服务，需要分配新的端口
- 在Nginx网关中添加新的路由规则
- 部署容器时需要自定义容器名或网络名
- 手动创建数据库或MinIO桶

#### 占用登记流程（三步完成）

```
第1步：查附表
  └─ 在对应附表中确认你要占用的资源是否可用
      判断标准：该行的"占用项目"列不为空，"释放日期"列为空或为"—"

第2步：登记
  └─ 在附表中新增一行，填写：
      资源名称 | 你的项目编号 | 用途说明 | 日期 | 你的名字 | 释放日期留空 | 释放人留空 | 备注

第3步：升版
  └─ 修改文件头部的版本号：将 r 后面的数字加 1
      （例如当前版本 v1.1.0r5 → 你登记后改为 v1.1.0r6）
  └─ 在版本历史表中追加一行记录
```

#### 释放登记流程（三步完成）

```
第1步：找到之前的占用行
  └─ 在对应附表中找到你之前占用的那一行

第2步：登记释放
  └─ 在该行的"释放日期"列填入当前日期
  └─ 在该行的"释放人"列填入你的名字

第3步：升版
  └─ 修改文件头部的版本号：将 r 后面的数字加 1
  └─ 在版本历史表中追加一行记录
```

#### 占用/释放示例（端口登记）

| 端口 | 占用项目 | 服务名 | 用途 | 占用日期 | 负责人 | 释放日期 | 释放人 | 备注 |
|:---|:---|:---|:---|:---|:---|:---|:---|:---|
| (举例) 30200 | (举例) w003 | (举例) w003-payment-api | (举例) 支付服务HTTP | (举例) 2026-05-11 | (举例) 张三 | — | — | 举例，资源未被实际占用 |
| (举例) 30201 | (举例) w003 | (举例) w003-refund-api | (举例) 退款服务HTTP | (举例) 2026-05-15 | (举例) 张三 | (举例) 2026-06-01 | (举例) 张三 | 举例，资源未被实际占用 |

- 第1行：释放日期为"—"，表示仍在占用，其他人不可使用端口30200
- 第2行：释放日期为"2026-06-01"，表示已于该日释放，其他人可以申请使用端口30201

#### 违反后果

- 未登记导致的资源冲突（端口冲突、路由冲突等），由相应团队自行承担故障责任和修复工作
- 平台管理员发现未登记的资源占用，有权强制停止相关服务

### 4.1 创建新项目

每个新项目通过 `create-project.sh` 脚本自动化入驻，脚本会自动完成：

1. 分配100个端口（30000起步，每项目+100）
2. 创建独立的 PostgreSQL 数据库 + 用户
3. 创建独立的 Redis ACL 用户（带key前缀隔离）
4. 创建独立的 MinIO 存储桶
5. 生成项目目录和 docker-compose.yml 模板
6. 将凭证写入 `.env.secrets`

```bash
platformctl project create <项目编号> <项目描述名>

# 示例
platformctl project create w001 user-service
platformctl project create w002 order-service
```

> 项目创建后，请按照第4.0节要求在附表A-F中登记占用的资源。

### 4.2 项目目录结构

```
projects/
└── w001-user-service/
    ├── docker-compose.yml    # 项目服务编排（已预填基础设施连接信息）
    ├── Dockerfile            # Go/Gin 多阶段构建模板
    ├── configs/              # 项目自定义配置
    └── db-migrations/        # 数据库迁移脚本
```

### 4.3 部署你的代码

```bash
cd /opt/neuro-devsituat-platform/projects/w001-user-service
git clone <你的仓库地址> .
# 修改 docker-compose.yml
docker compose up -d
```

> 部署前请先在第4.0节对应附表中登记端口和容器名。

### 4.4 配置 API 网关路由

编辑 `/opt/neuro-devsituat-platform/configs/nginx/nginx.conf`：

```nginx
upstream w001_user_service {
    server w001-user-service:30000;
}

# 在 server { } 块中添加
location /api/v1/user/ {
    proxy_pass http://w001_user_service/api/v1/user/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

重载：`docker exec gateway-nginx nginx -s reload`

> 添加路由后，请在附录B中登记路由路径。

## 8. 故障排查

```bash
# 服务无法启动
docker ps -a
docker logs <容器名> --tail 50
free -h && df -h && docker stats --no-stream

# 数据库连接失败
docker exec postgresql18 pg_isready -U platform_admin
docker exec postgresql18 psql -U platform_admin -c "\l"

# Redis连接失败
docker exec redis7 redis-cli -a <密码> ping    # 应返回 PONG

# 磁盘不足
df -h && docker system prune -a -f

# 端口冲突
ss -tlnp | grep <端口号>
```

## 9. 安全注意事项

1. **凭证管理**: 密码存储在 `.env.secrets`（权限600），禁止明文写入代码
2. **环境隔离**: 严禁接入生产流量或存储生产敏感数据
3. **端口规范**: 基础设施 20000-29999，项目服务 30000-49999
4. **资源限制**: 每个容器设有CPU和内存上限
5. **数据备份**: PostgreSQL每日凌晨2点自动备份，保留7天
6. **日志轮转**: Docker容器日志自动轮转（单容器上限30MB）
7. **资源登记**: 部署新服务前必须在附录A-F中登记资源占用，释放后也需在表中标记

## 附录H: Dockerfile 模板

```dockerfile
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o main .

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/main .
EXPOSE 8080
USER nobody
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --spider -q http://127.0.0.1:8080/health || exit 1
ENTRYPOINT ["./main"]
```
